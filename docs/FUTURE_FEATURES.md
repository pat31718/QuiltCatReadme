## 1. 寵物分支成就系統 ✅ 已實作

> **狀態：已於 Pet Feature Upgrade 1.0 中實作完成**

### 實作內容

- 新增 `cat pet collect` 指令，顯示寵物圖鑑
- 已收集的寵物顯示種類名稱、當時的寵物名稱、首次獲得時間
- 未收集的寵物以「❓ ??? (未知種類)」顯示
- 支援收集進度統計（例如「已收集 5/10 種」）
- 觸發記錄的時機：領養新寵物、進化、放生、等級回退
- 新增 `CollectedPetRecord` 類別與 `PetBox` 類別
- 舊版資料自動遷移（現有寵物自動記錄到圖鑑）

## 2. 寵物重置機制 ✅ 已實作

> **狀態：已於 Pet Feature Upgrade 1.0 中實作完成**

### 實作內容

**放生（`cat pet release`）：**
- 免費放生目前的寵物
- 放生前自動記錄到圖鑑
- 若收納盒有寵物，自動取出第一隻作為跟隨中寵物
- 需回答 yes/no 進行二次確認

**等級回退（`cat pet reset`）：**
- 消耗對應屬性水果將寵物退回上一等級
- 回退後存活時間與所有屬性點數重置為零
- 回退前後的寵物種類都記錄到圖鑑
- 消耗水果數量：LV1→Basic 5 顆 / LV2→LV1 20 顆 / LV3→LV2 50 顆
- 根據寵物屬性消耗對應水果（INT→檸檬、DEX→葡萄、STR→櫻桃）

## 3. 多寵物飼養系統（寵物收納盒） ✅ 已實作

> **狀態：已於 Pet Feature Upgrade 1.0 中實作完成**

### 實作內容

- 新增 `PetBox` 類別取代直接使用 `PetData`
- `petDataDic` 型別從 `Dictionary<userId, PetData>` 改為 `Dictionary<userId, PetBox>`
- 新增 `PetBoxConverter` 實現舊版 JSON 資料自動遷移（向後相容）
- 解鎖條件：任一寵物達到系統最高等級（目前為稀有 LV2），解鎖 1 格收納盒
- `cat pet new`：若收納盒有空間，目前寵物自動移入收納盒
- `cat pet box`：查看收納盒中的寵物
- `cat pet swap <編號>`：交換跟隨中與收納盒中的寵物
- `cat feed`：收納盒中的寵物也會一併餵食（需有足夠飼料）

## 4. 股票交易系統

### 構想

加入類似《動物森友會》大頭菜系統的股票交易機制。每週的股票價格會依照預先決定的模型波動，使用者可以在低價買入、高價賣出。

**核心設計：**
- 使用小魚乾作為**進場手段**（購買股票用小魚乾支付）
- 獲利以**新貨幣或點數**發放（參見 [第二貨幣與點數系統](#5-第二貨幣與點數系統)）
- 週末過期：尚未賣出的股票在週末歸零，投入的小魚乾損失，藉此維持系統的常駐參與價值

**主要功能：**
- 新增股票市場指令群（例如 `cat stock`）
- 每週初（例如星期一）決定當周的價格模型
- 每天（或每半天）價格會按照模型變化
- 使用者可以購買和賣出股票
- 週末過期：未賣出的持股歸零，投入的小魚乾損失

### 指令規劃

| 指令 | 說明 |
|------|------|
| `cat stock` | 顯示目前的股價與持有數量 |
| `cat stock buy <數量>` | 以目前價格購買股票（消耗小魚乾） |
| `cat stock sell <數量>` | 以目前價格賣出股票（獲得新貨幣/點數） |
| `cat stock history` | 查看本周的歷史股價走勢 |

### 價格模型設計

參考動物森友會的大頭菜模型，可設計以下幾種模式：

| 模型 | 說明 | 出現機率 |
|------|------|---------|
| 波動型 | 價格上下震盪，有高有低 | 35% |
| 遞減型 | 價格持續下降，虧損機率高 | 25% |
| 三期型 | 先降後升再降，中間有一次高峰 | 25% |
| 四期型 | 先降後持續上升，末段大漲 | 15% |

**價格範圍建議：**
- 基礎價格：100 小魚乾 / 股
- 最低價格：約 40 小魚乾 / 股（遞減型末段）
- 最高價格：約 600 小魚乾 / 股（四期型高峰）

### 週末過期機制

- 每週日午夜 0 時，所有尚未賣出的持股**自動歸零**，投入的小魚乾視為損失
- 不做自動結算，未賣出就是虧損——這樣才能讓股票系統保有常駐的參與價值
- 使用者必須在週末前自行決定賣出時機，增加策略深度與緊張感
- 系統會在接近截止時間時發送提醒通知（例如週六晚間提醒尚有未賣出的持股）

### 資料結構

```
StockData (新增類別，伺服器層級)
├── currentModel: StockModel          → 當周的價格模型
├── modelStartDate: DateTime          → 模型開始日期
├── priceHistory: List<decimal>       → 歷史價格紀錄
├── currentPrice: decimal             → 目前股價
└── nextPriceChangeTime: DateTime     → 下次股價更新時間

UserStockData (新增類別，使用者層級)
├── holdings: int                     → 持有股數
├── averageCost: decimal              → 平均持股成本
└── totalInvested: decimal            → 總投入金額（小魚乾）
```

### 優點

- 使用小魚乾進場，提供元老使用者消耗大量小魚乾的管道
- 社群互動性高（討論股價、分享買賣時機）
- 高風險高報酬的機制，增加遊戲刺激感
- 與現有系統完全獨立，不會影響既有功能
- 每週不同模型讓遊戲保持新鮮感
- 週末過期機制增加緊張感與策略深度，鼓勵使用者持續關注

### 缺點

- 開發工作量大（需新增完整的資料模型、指令、排程系統）
- 價格模型的平衡性需要長時間調整
- 需要穩定的背景排程來更新股價（定時任務的可靠性）
- 需要搭配第二貨幣系統才能避免小魚乾經濟失衡

## 5. 第二貨幣與點數系統

### 背景問題

目前有一個隱性的經濟問題：元老使用者因為長期累積的銀行利息，持有大量的小魚乾，缺乏有趣的消費管道。  
如果所有新功能的獎勵都繼續發放小魚乾，會加劇通貨膨脹問題。

### 構想

新增一種不同於小魚乾的貨幣或點數系統：
- **小魚乾**作為「進場手段」——用來參與新功能（例如購買股票、進入活動）
- **新貨幣/點數**作為「獎勵產出」——從新功能中獲得的報酬是新貨幣，不是小魚乾

這樣設計的好處是：小魚乾有了新的消耗管道（進場花費），同時新功能的獎勵不會讓小魚乾繼續膨脹。

### 可能的新貨幣形式

| 名稱方案 | 概念 | 用途 |
|---------|------|------|
| 珊瑚幣 | 進階貨幣，只能透過新功能獲取 | 兌換限定道具、寵物裝飾、特殊稱號 |
| 探險點數 | 累積型點數，不可交易 | 解鎖寵物探險獎勵、兌換稀有道具 |
| 星塵 | 稀有貨幣，產出量少 | 購買最高級道具、解鎖隱藏功能 |

### 適用場景

| 場景 | 進場消耗（小魚乾） | 獎勵產出（新貨幣） |
|------|-------------------|-------------------|
| 股票交易 | 購買股票 | 賣出獲利以新貨幣計算 |
| 寵物探險 | 派出探險消耗補給品 | 探險成功獲得點數 |
| 季度活動 | 活動入場券 | 活動獎勵為限定兌換幣 |
| 寵物戰鬥（未來） | 報名參賽費 | 勝利獎勵為新貨幣 |

### 優點

- 解決元老使用者小魚乾過多的問題（提供消耗管道）
- 新功能的獎勵不會加劇小魚乾通膨
- 新貨幣可用於兌換獨特的獎勵，提供新的收集目標
- 經濟系統更有層次感（基礎貨幣 + 進階貨幣）

### 缺點

- 需要設計新貨幣的儲存、顯示與兌換邏輯
- 兩種貨幣可能讓新手困惑
- 新貨幣的價值需要長期調整與平衡
- 兌換獎勵的內容需要持續更新才能維持吸引力

## 6. 寵物探險系統

### 構想

新增「寵物探險」功能，讓使用者可以派寵物出去探險。探險需要一段時間（例如數小時），回來後可以獲得獎勵。這個功能可以作為新貨幣/點數的主要產出方式之一。

**主要功能：**
- 使用 `cat pet explore` 派出寵物去探險
- 探險需要等待一段時間（例如 4~8 小時）
- 探險結果根據寵物的屬性（INT / DEX / STR）影響獎勵品質
- 探險期間寵物仍可餵食、查看，但不能進化
- 可消耗小魚乾提升探險品質或縮短時間（不使用現有道具）

### 指令規劃

| 指令 | 說明 |
|------|------|
| `cat pet explore` | 派出寵物探險（若已在探險中則顯示剩餘時間） |
| `cat pet explore check` | 查看探險進度與預計回來時間 |
| `cat pet explore collect` | 探險結束後領取獎勵 |

### 探險獎勵

探險獎勵**完全與現有道具系統切開**，不會產出任何現有道具（水果、道具等）。  
這樣可以避免獎勵系統混亂，並讓新貨幣有明確的產出管道。

| 獎勵類型 | 說明 |
|---------|------|
| 新貨幣/點數 | 唯一的實質獎勵，數量依據寵物等級與屬性 |
| 探險日誌 | 每次探險產生一段隨機短故事文字，增加趣味與社群分享性 |

> **設計原則**：探險獎勵只產出新貨幣/點數，不與現有的小魚乾或道具系統重疊。

### 優點

- 不需要使用者全程在線，掛機型功能適合忙碌的使用者
- 讓寵物的屬性數值有更多實際用途
- 可作為新貨幣的穩定產出來源
- 探險日誌的隨機文字可增加社群趣味

### 缺點

- 需要背景排程處理探險時間計算
- 獎勵數值需要平衡，避免掛機產出過多
- 探險期間寵物狀態的處理需要額外邏輯

## 7. 其他可能的擴充方向

以下是一些可以進一步討論的功能方向：

### 寵物戰鬥系統
- 使用者之間的寵物對戰
- 利用寵物的 INT / DEX / STR 屬性進行回合制戰鬥
- 勝利者可獲得新貨幣/點數獎勵（非小魚乾）
- **適合搭配**：多寵物系統、LV3 進化開放

### 寵物 LV3 / LV4 開放
- 目前 LV3 的寵物種類已在程式碼中定義但尚未完全開放
- LV4 等級已在枚舉中預留
- 可設計更稀有的進化條件
- **適合搭配**：成就系統（增加收集目標）

### 伺服器排行榜
- 寵物收集排行榜（收集最多種類的使用者）
- 簽到排行榜（最高連續簽到天數）
- 探險排行榜（累計探險次數或獲得最多新貨幣）
- 可每週或每月重置，增加競爭性

> **注意**：不設置小魚乾排行榜，因為資產排名涉及個人隱私。

### 季度事件系統
- 每一季舉辦一次大型活動，設定全伺服器共同目標
- 例如：全員累計工作次數達標、全員累計餵食寵物次數達標
- 達標後全伺服器獲得獎勵（新貨幣/點數，非小魚乾）
- 限定道具（只在特定活動期間可購買或獲得）
- 需要有新的獎勵內容（例如限定寵物裝飾、稱號等）來支持誘因

## 開發注意事項

1. **經濟平衡**：新增第二貨幣後，需要同時管理兩種貨幣的產出與消耗比例
2. **小魚乾消耗**：新功能應優先考慮「消耗小魚乾」的設計方向，讓元老使用者有地方花費
3. **資料遷移**：修改資料結構時，必須考慮舊版資料的向後相容性
4. **使用者體驗**：新增指令時需保持操作簡潔，避免過於複雜的參數
5. **伺服器隔離**：所有新功能都應維持各伺服器資料獨立的設計原則
6. **錯誤處理**：新功能需統一使用 `ErrorLogWriter.WriteLog()` 記錄錯誤
7. **獎勵設計**：需要持續規劃新貨幣可兌換的獎勵內容，維持使用者的收集動力
